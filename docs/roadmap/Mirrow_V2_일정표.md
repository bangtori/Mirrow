# Mirrow V2.0.0 로드맵

> 목표: 외부 유입(STILLSEE 계정 홍보)을 시작하기 전, 데이터 수집 기반을 다지고 UI를 외부인에게 보여줄 수 있는 수준으로 개편한다.
> 배포 방식: Vercel 자동배포 — 작업 단위별로 자주 배포.

## v1.0.2 — 이벤트 로그 확장

가장 먼저 진행. 이후 모든 작업의 데이터가 여기서부터 쌓이기 시작하므로 지연될수록 손실 기간이 길어짐.

### 체크리스트
- [ ] `LANDING_VIEWED` 이벤트 추가 (랜딩 페이지 진입 시)
- [ ] 이벤트 메타데이터 구조에 `source` 필드 추가 (예: `{ source: "instagram" }`)
  - [ ] URL 쿼리 파라미터(예: `?ref=instagram`) → 메타데이터 매핑 로직 설계
  - [ ] 기존 이벤트(`PROFILE_CREATED`, `RESPONSE_LINK_COPIED` 등)에도 `source` 적용 검토
- [ ] `PROFILE_CREATE_STARTED` 이벤트 추가 (이름 입력 후 "다음" 클릭 시)
- [ ] `RESPONSE_PAGE_VIEWED` 이벤트 추가 (응답 링크 진입 시)
- [ ] 이벤트 상수 파일(`constants`) 업데이트 및 타입 정의
- [ ] Supabase `events` 테이블에 정상 적재되는지 로컬/스테이징 확인
- [ ] **배포: `v1.0.2`**
