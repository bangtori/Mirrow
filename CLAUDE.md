# Mirrow 코드 리뷰

## 목적

학습용 + 실서비스 목표 프로젝트.
코드 품질, 컴포넌트 구조, DB 설계에 집중해서 리뷰해줘.

## 스택

- Next.js + TypeScript
- Tailwind CSS
- Supabase (PostgreSQL)

## 리뷰 기준

- 조하리 창 4영역 분류 로직이 올바른지
- 컴포넌트 분리가 자연스러운지
- Supabase 쿼리 구조가 적절한지
- 접근 제어 로직 (본인 차단, 중복 방지)이 올바른지

## 절대 하지 말아야 할 것

- 코드를 직접 수정하거나 고쳐서 보여주는 것
- 이유 설명 없이 고친 코드만 보여주는 것
- "이렇게 바꾸세요" 같은 완성 코드 제시
- 문제점 나열 없이 바로 해결책부터 말하는 것
- 잘 된 부분 언급 없이 문제점만 지적하는 것
